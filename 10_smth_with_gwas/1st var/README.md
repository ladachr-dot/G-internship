Этот инструмент предназначен для первичной обработки экспортных данных из GWAS Catalog и подготовки поисковых запросов для баз данных (например, NCBI PubMed или ClinVar).

---

## 📖 Руководство по работе с GWAS-парсером (`gen1.1.py`)

### Предназначение скрипта

Скрипт автоматизирует работу с TSV-файлами из GWAS Catalog.

**Что делает скрипт:**

* **Работа с пропусками:** Находит столбцы с rsID, OR, Beta-коэффициентами и RAF.
* **Очистка данных:** Извлекает чистые числовые значения из строк.
* **Генерация запроса:** Создает готовую строку для вставки в поиск NCBI.
---

### 1. Подготовка данных

Перед запуском убедитесь, что:

1. Файл сохранен в формате **.tsv**.
2. В файле присутствуют стандартные заголовки GWAS Catalog (например, `riskAllele`, `OR`, `beta`, `riskFrequency`).

---

### 2. Инструкция по запуску

#### Шаг 1: Окружение

Убедитесь, что у вас установлены библиотеки `pandas` и `numpy`.

#### Шаг 2: Запуск

Запустите скрипт через терминал или IDE-

#### Шаг 3: Ввод параметров

1. **Путь к файлу:** Введите полное имя файла (например, `gwas_results.tsv`).
2. **Количество rsID:** Укажите, сколько вариантов с высокой частотой (RAF) нужно для запроса. По умолчанию — **20**.

---

### 4. Результаты 

После выполнения скрипт создаст два файла в той же папке:

1. **`ncbi_query.txt`**: Текстовый файл со списком rsID, разделенных «OR».
* *Как использовать:* Скопируйте содержимое и вставьте в строку поиска на [PubMed](https://pubmed.ncbi.nlm.nih.gov/).

2. **`top_variants_sorted_by_RAF.csv`**: Отфильтрованная таблица, где сверху находятся самые часто встречающиеся в популяции варианты.
