## Руководство

Настоящее руководство описывает порядок работы с обновленным программным модулем gen.py. В отличие от предыдущей версии, данный скрипт интегрирует данные из API Ensembl для обогащения первичной выборки GWAS клинической значимостью и популяционными частотами.

---

1. Подготовка к работе

Для успешной эксплуатации модуля необходимо:
- Среда исполнения: Python 3.x с установленными зависимостями: pandas, numpy, requests, tabulate.
- Входной файл: Таблица в формате .tsv с заголовками, соответствующими номенклатуре GWAS Catalog (например, variantId, OR, beta, riskFrequency).
- Сетевое соединение: Активное интернет-соединение для обмена данными с сервером Ensembl.

2. Алгоритм запуска и эксплуатации

Шаг 1: Инициализация
Запустите скрипт через терминал:

Bash
python gen.py

Шаг 2: Параметризация
Система запросит ввод следующих данных:

Имя файла: Введите путь к TSV-файлу. Скрипт поддерживает поиск в текущем каталоге и папке «Загрузки».
Объем выборки: Укажите количество топовых вариантов для детального анализа (по умолчанию — 20).

Шаг 3: Обработка
Скрипт последовательно выполнит следующие действия:

- Валидация данных и очистка от пропусков.
- Расчет скорингового показателя для ранжирования.
- Запрос к API Ensembl с автоматической обработкой ограничений скорости (HTTP 429).

3. Результаты
После успешного завершения операции создаются следующие файлы:

- genomics_report.csv: Итоговая таблица, содержащая исходные параметры и дополненная аннотациями из Ensembl.
- ncbi_query.txt: Сформированный запрос для NCBI, объединяющий отобранные варианты оператором «OR».

---
Техническое примечание: При большом количестве запросов (параметр top_n) время выполнения может увеличиться из-за регламента работы API-сервера Ensembl.
