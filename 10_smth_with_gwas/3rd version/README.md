## Руководство пользователя
Настоящий документ описывает функционал модуля itogg.py, который расширяет возможности первичной обработки геномных данных за счет автоматизированной интеграции с Google Sheets API.

1. Ключевые возможности

- Аналитическая фильтрация: Ранжирование вариантов по статистическим показателям (OR/Beta).
- Аннотация через Ensembl: Обогащение данных клинической значимостью и частотами аллелей (европейские популяции: CEU, FIN, GBR, IBS, TSI).
- Облачная синхронизация: Автоматическая выгрузка структурированных результатов в Google Таблицы.

2. Требования к подготовке
Для успешной работы модуля необходимо:

1. Файл: Наличие credentials.json в той же директории, что и скрипт.
2. Создайте таблицу на Вашем Google Диске
3. Откройте доступ к таблице для 'gene-885@peppy-winter-489319-b1.iam.gserviceaccount.com' в качестве редактора
4. Наличие установленных библиотек: pandas, numpy, requests, tabulate, google-auth, google-api-python-client, gspread oauth2client
   
3. Инструкция по эксплуатации
Шаг 1: Инициализация и обработка
После запуска скрипта и ввода параметров (имя TSV-файла, объем выборки) система выполнит:

- Обработку данных и формирование локального отчета genomics_report.csv.
- Генерацию файла для поискового запроса ncbi_query.txt.

Шаг 2: Выгрузка в Google Sheets
После завершения расчетов скрипт затребует следующие параметры:

- Название таблицы: Полное имя целевой Google Таблицы на вашем диске.
- Название листа: Имя конкретного листа, куда будут внесены данные. Если лист отсутствует, система автоматически создаст его.

> Примечание: При повторном запуске программы с теми же данными таблицы значения на Google Диске обновляются

4. Особенности реализации
Очистка данных: Система автоматически заменяет значения NaN на пустые строки для корректной передачи в API Google Sheets.
Управление лимитами: Встроенная задержка (1 сек) при ответе сервера 429 обеспечивает стабильное взаимодействие с API Ensembl.
Перезапись данных: Функция worksheet.clear() удаляет предыдущие записи, обеспечивая актуальность отчета в облаке при каждом запуске.
